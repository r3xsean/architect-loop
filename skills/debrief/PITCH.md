# Pitch extension — buy-in material

Reviewers and leadership inherit the unknowns the author started with. The pitch pre-answers them so approval doesn't wait on rediscovery.

Extend the debrief report (or produce a standalone shareable HTML) with:

- **The demo up top** — a GIF, recording, or interactive mock of the change working. Lead with it; nothing before it.
- **The objections pre-answered** — every concern a reviewer or expert would raise (the failure points, the alternatives considered, the deviations taken), each with the evidence that settles it.
- **Sign-offs named** — exactly who needs to approve what, so the doc routes itself.

Written to drop into Slack or a PR description as one link; self-contained, no repo access needed to read it.
